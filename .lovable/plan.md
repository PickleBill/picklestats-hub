

# Pickle DaaS — ElevenLabs TTS + Lovable Cloud + Firecrawl Integration

## Overview

Three integrations in sequence: (1) Connect ElevenLabs and build a TTS edge function replacing browser `speechSynthesis`, (2) Enable Lovable Cloud with Supabase database for real data, (3) Connect Firecrawl for brand scraping.

---

## Step 1: Connect ElevenLabs

Use Lovable's native ElevenLabs connector to securely inject your API key. You already have a Creator account — we'll link it here so the `ELEVENLABS_API_KEY` is available in edge functions automatically.

## Step 2: Enable Lovable Cloud

Enable Lovable Cloud so we get a Supabase backend. This gives us edge functions, a Postgres database, and the `LOVABLE_API_KEY` (already present).

## Step 3: Create Database Schema

Run migrations to create three tables from your overnight build's schema:
- `pickle_daas_analyses` — clip metadata, scores, commentary, brands, badges
- `pickle_daas_brands` — brand detection registry
- `pickle_daas_player_dna` — player profiles with aggregated stats

Seed with the 3 existing clips + PickleBill player data + brand entries.

## Step 4: Build ElevenLabs TTS Edge Function

Create `supabase/functions/elevenlabs-tts/index.ts`:
- Accepts `{ text, persona }` (espn, hype, ronBurgundy, chuckNorris)
- Maps each persona to a distinct ElevenLabs voice ID:
  - ESPN → "Brian" (nPczCjzI2devNBz1zQrb) — authoritative announcer
  - Hype → "Eric" (cjVigY5qzO86Huf0OWal) — energetic
  - Ron Burgundy → "George" (JBFqnCBsd6RMkjVDRZzb) — smooth, dramatic
  - Chuck Norris → "Chris" (iP95p4xoKVk53GoZ742B) — deep, deadpan
- Returns binary MP3 audio
- Uses `eleven_turbo_v2_5` model for low latency

## Step 5: Update Frontend — Replace speechSynthesis

Update `CommentaryTabs.tsx` and `VoiceLab.tsx`:
- Replace `speechSynthesis` calls with `fetch` to the TTS edge function
- Play returned audio blob via `new Audio(URL.createObjectURL(blob))`
- Add loading state on the Play button while audio generates
- Keep `speechSynthesis` as fallback if edge function fails

## Step 6: Wire App to Supabase Data

- Create `src/integrations/supabase/` client setup (auto-generated by Cloud)
- Create `src/hooks/useClips.ts` — fetches from `pickle_daas_analyses`
- Create `src/hooks/usePlayerDNA.ts` — fetches from `pickle_daas_player_dna`
- Create `src/hooks/useBrands.ts` — fetches from `pickle_daas_brands`
- Update Dashboard, Highlights, PlayerDNA, Brands, VoiceLab pages to use React Query hooks instead of `src/data/clips.ts`
- Keep hardcoded data as fallback when DB is empty

## Step 7: Connect Firecrawl

Use Lovable's Firecrawl connector. Build an edge function `firecrawl-scrape` for brand logo/data enrichment on the Brands page.
